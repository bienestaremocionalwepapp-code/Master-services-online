🔄 Flujo de verificación
1. Inicio del cliente → verifica contra este servidor cada 24h.
2. Con conexión → respuesta en tiempo real (ACTIVA / SUSPENDIDA / VENCIDA / BLOQUEADA).
3. Sin internet → usa caché local (válido 72h desde última verificación exitosa).
4. Caché expirado + sin conexión → bloqueo por CACHE_EXPIRED.
5. Licencia PERPETUA → activada con PerpetualKey, nunca más necesita servidor.
📋 Estados de licencia
ACTIVAFuncionamiento normal. Puede acceder a la app.
SUSPENDIDABloqueada por falta de pago. Muestra mensaje al cliente.
VENCIDAExpiró el período + días de gracia. Auto-marcada por CRON diario.
BLOQUEADABloqueo total inmediato sin mensaje (uso: fraude, disputa).
PERPETUASin vencimiento. Puede operar offline con PerpetualKey.
GRACIAPeríodo de gracia activo. Acceso con advertencia de renovación.
⚙️ Endpoints de la API
GET ?action=verify&key=SRVT-XXXX...&domain=X&fingerprint=X → Verificar (público)
GET ?action=list&token=TU_TOKEN → Listar licencias
GET ?action=create&token=T&client=X&plan=MENSUAL → Crear licencia
GET ?action=suspend&key=X&token=T&reason=Y → Suspender
GET ?action=activate&key=X&token=T → Reactivar
GET ?action=block&key=X&token=T → Bloqueo total
GET ?action=mkperp&key=X&token=T → Convertir a perpetua
GET ?action=panel&token=T → Panel HTML (este)
🔧 Setup del panel en Apps Script
1. Crea un Google Spreadsheet vacío (tu panel maestro).
2. Extensiones → Apps Script → pega LicenseMaster.gs.
3. Cambia MASTER_SECRET y ADMIN_TOKEN por valores únicos.
4. Ejecuta setupMaster() una sola vez.
5. Implementa como Web App → Ejecutar como: Yo / Acceso: Cualquiera.
6. La URL generada es tu LICENSE_SERVER_URL.
7. Instala trigger diario: ejecuta installDailyTrigger().
